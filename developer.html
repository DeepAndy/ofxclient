class ofxclient.Account(number, institution, description=None)¶

Base class for accounts at an institution

It provides basic methods for retrieving transactions that are common to all different types of accounts.

Subclasses only need to implement the download_query method which is expected to take an ‘as_of’ parameter representing an effective lower date boundary in ‘YYYYMMDD’ format. The return value of this function is expected to be a prepared query as returned by the ofxclient.client.Client

See these subclasses for more details: BankAccount CreditCardAccount BrokerageAccount

class ofxclient.Institution(id, org, url, username, password, broker_id='', description=None, client_args={})¶

Represents an institution or bank

This is where you specify all connection details needed for the specific institution.

For help obtaining the id, org, and url; please see the ofxhome python module and/or the http://ofxhome.com website.

An optional description can be passed in for, well, descriptive purposes.

The last optional parameter, client_args, should be a dictionary containing some or all of the parameters that ofxclient.client.Client takes. This allows you to override the settings for a particular institution

class ofxclient.config.SecurableConfigParser(keyring_name='ofxclient', keyring_available=True, **kwargs)[source]¶

ConfigParse subclass that knows how to store options marked as secure into the OS specific keyring/keychain.

To mark an option as secure, the caller must call ‘set_secure’ at least one time for the particular option and from then on it will be seen as secure and will be stored / retrieved from the keychain.

get(section, option, *args)[source]¶

If an option is secure, return the plain text from the secure storage backend. Otherwise this acts just like ConfigParser.get

has_secure_option(section, option)[source]¶

Like ‘has_option’ but also tests whether the option is secure.

is_secure_option(section, option)[source]¶

return True if the option is secure, False otherwise.

items(section)[source]¶

Subclassed items method which knows how to detect secured options and will retrieve the information in the clear from the secure storage backend

remove_option(section, option)[source]¶

Removes the option from ConfigParser as well as the secure storage backend>

secure_items(section)[source]¶

Return a list of options marked as secure for the given section. See the documentation for ‘ConfigParser.items’ for more information.

set(section, option, value)[source]¶

Just like ConfigParser set except that it knows about secure options.

set_secure(section, option, value)[source]¶

Set an option and mark it as secure.

Any subsequent uses of ‘set’ or ‘get’ will also now know that this option is secure as well.

write(*args)[source]¶

Write a-la ConfigParse but also writes out any unsaved changes to the secure storage backend.
