class ofxclient.Account(number, institution, description=None)¶

Base class for accounts at an institution

It provides basic methods for retrieving transactions that are common to all different types of accounts.

Subclasses only need to implement the download_query method which is expected to take an ‘as_of’ parameter representing an effective lower date boundary in ‘YYYYMMDD’ format. The return value of this function is expected to be a prepared query as returned by the ofxclient.client.Client

See these subclasses for more details: BankAccount CreditCardAccount BrokerageAccount

class ofxclient.Institution(id, org, url, username, password, broker_id='', description=None, client_args={})¶

Represents an institution or bank

This is where you specify all connection details needed for the specific institution.

For help obtaining the id, org, and url; please see the ofxhome python module and/or the http://ofxhome.com website.

An optional description can be passed in for, well, descriptive purposes.

The last optional parameter, client_args, should be a dictionary containing some or all of the parameters that ofxclient.client.Client takes. This allows you to override the settings for a particular institution

accounts()¶

Return a list of ofxclient.Account objects for this institution

These objects let you download statements, transactions, positions, and perform balance checks.
